# 🧩 트랜스미디어 콘텐츠 캐릭터 플랫폼 Use Case Document

---

## 📘 1. Actor Definitions (행위자 정의)

| 행위자                         | 설명                                       | 주요 권한/행동                                                |
| --------------------------- | ---------------------------------------- | ------------------------------------------------------- |
| **일반 사용자(User)**            | 웹사이트에 로그인하여 캐릭터 콘텐츠를 탐색, 참여, 구매하는 주요 이용자 | - 캐릭터 정보 열람<br>- 챌린지 참여<br>- 팝업스토어/굿즈 구매<br>- 투어 스탬프 수집 |
| **관리자(Admin)**              | 콘텐츠와 이벤트를 관리하고, 사용자 참여 데이터를 모니터링         | - 캐릭터 정보 등록/수정<br>- 이벤트 관리<br>- 통계 데이터 확인               |
| **게스트(Guest)**              | 로그인하지 않은 방문자                             | - 기본 캐릭터 정보 열람<br>- 회원가입/로그인 유도                         |
| **결제 시스템(Payment Gateway)** | 외부 결제 연동 시스템                             | - 결제 승인/취소/환불 처리                                        |
| **브랜드 파트너(Brand Partner)**  | 팝업스토어나 이벤트 협업 브랜드                        | - 이벤트 연계 정보 제공<br>- 굿즈 등록 승인                            |

## ⚙️ 3. Main Steps (주요 단계)

### UC-01. 캐릭터 소개 열람

1. 사용자가 메인 페이지 접속
2. “캐릭터 소개” 메뉴 클릭
3. 캐릭터 목록 표시 (썸네일 + 이름)
4. 특정 캐릭터 선택 → 상세 페이지 진입
5. 캐릭터 페르소나, 스토리, 관련 미디어 표시

---

### UC-05. 굿즈 구매

1. 사용자 로그인 확인
2. “굿즈샵” 메뉴 진입
3. 상품 목록에서 항목 선택
4. 상세 페이지에서 “구매하기” 클릭
5. 결제 정보 입력 후 결제 요청 → 결제 게이트웨이 호출
6. 결제 승인 → 주문 완료 메시지 표시

## 🧭 4. Exception Handling (예외 처리)

| 상황         | 원인          | 처리 방식                   |
| ---------- | ----------- | ----------------------- |
| 로그인 세션 만료  | 일정 시간 미사용   | 로그인 페이지로 리다이렉트          |
| 결제 실패      | 결제 게이트웨이 오류 | “결제 실패” 메시지 표시 및 재시도 유도 |

| 인증되지 않은 접근 | 비로그인 사용자 접근 | “로그인이 필요합니다” 알림창 표시     |

## 🎬 6. Detailed Use Case Scenarios (상세 시나리오)

### UC-02. 캐릭터 챌린지 참여

**Preconditions:**

* 사용자가 로그인 상태
* 챌린지 이벤트 기간 내

**Flow of Events:**

1. 사용자가 챌린지 메뉴 클릭
2. 참여 가능한 챌린지 리스트 출력
3. 챌린지 선택 → 상세 설명 및 참여 버튼 노출
4. 사용자가 미션 수행 (예: 웹툰 감상, 스토리 댓글 작성)
5. 미션 완료 시 ‘리워드 알림’ 표시

**Alternative Flow:**

* 로그인하지 않은 상태 → 로그인 창으로 이동
* 이미 참여한 챌린지 → “이미 참여한 챌린지입니다” 표시

**Postconditions:**

* 사용자 포인트 적립
* 마이페이지의 “참여 기록”에 추가
